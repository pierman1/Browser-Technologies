Fallbacks

Als <input type="date"> niet supported is ziet de browser: <input type="text">. Je kan een placeholder attribuut gebruiken:
<input type="date" placeholder="mm/dd/yy"> browsers die dit niet supporten maken hiervan: <input type="text" placeholder="mm/dd/yy">. Nu heeft hij een placeholder om de gebruiker feedback te geven over het gewenste 'format'. De server checked de input en als het nodig is maakt hij er dit van:



Als <input type="color"> niet gesupport wordt door de browser: <input type="text">. Je kan ervoor kiezen om de gebruiker de kleur te laten typen in het inputveld, dit is wel erg fout gevoelig daarom is het misschien slim om een dropdown aan te bieden met de kleuren (standaard kleuren bv) daarin.

De 'color input' is een: <select> met een aantal standaard kleuren. De value string wordt geconverteerd naar een hex-code.

Om dit mogelijk te maken, heb je de volgende html elementen nodig:


<input type="color" id="colorpicker">

<select id="colorpicker-fallback">
<option value="red">red
<option value="green">green
<option value="blue">blue
<option value="orange">orange
<option value="purple">purple
<option value="pink">pink
<option value="brown">brown
<option value="black">black
<option value="white">white
<option value="grey">grey
<select>

Met een paar simpele regels JavaScript kan je vervolgens checken of de input typeattribuut text is, en op basis daarvan de default of de fallback tonen

var colorpicker = document.querySelector('#colorpicker');
var colorpickerAttr = colorpicker.getAttribute('type');
var colorpickerFallback = document.querySelector('#colorpicker-fallback');

if(colorpickerAttr == text){
colorpicker.classList.add('hide');
colorpickerFallback.classList.remove('hide');
}
